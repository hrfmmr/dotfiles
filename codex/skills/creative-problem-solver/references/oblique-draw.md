# Technique: Oblique Draw (prompt → lever)

## One-liner
視点を素早く切り替える手法。斜めプロンプトを引き、1つ選んで具体レバー/制約に翻訳する。

## Use when
- 局所最適にハマっている。
- 重いプロセスなしで火花が欲しい。
- 60秒以内で新しい角度を得たい。

## Avoid when
- 深いトレードオフ解消が必要（TRIZ）。
- 既に選択肢が多く、収束したい。

## Inputs
- 短いoblique promptデッキ（または下のmini-deck）。

## Procedure (fast, 1–3 min)
1. プロンプトを4枚引く。
2. 「少しズレているが面白い」1枚を選ぶ。
3. レバー/制約に翻訳する。
   - 「これは具体的に何を変える示唆か？」
4. そのレバー下で3案作る。

## Procedure (full, 8–12 min)
1. 6枚引いて2枚選ぶ。
2. 各プロンプトを2通り解釈する。
   - 技術解釈 + 社会/プロセス解釈。
3. 各解釈から2レバー導出。
4. 収束: 最も強いAhaを生むレバーを残す。

## Mini-deck (copy/paste)
- obviousな手を逆にする。
- 1ステップ削る。
- 先に可逆性を作る。
- 計画を変え得る最小テストを選ぶ。
- throughputではなくボトルネックを移す。
- 作業単位を変える。
- 制約を1つ別の制約に置き換える。
- 他領域のパターンを借りる。
- 努力量ではなく調整量を減らす。
- 安価なシグナルを1つ足す。
- 安全な選択をデフォルトにする。
- クリティカルパス外へ仕事を移す。

## Prompt bank (translation helpers)
- 「“1ステップ削る”なら、何を消して、代わりにどのガードレールを置くか？」
- 「“作業単位を変える”なら、新しい単位は何か？」
- 「“先に可逆性を作る”なら、rollback/kill switchは何か？」

## Outputs (feed CPS portfolio)
- 選択肢空間を作り直す1つのレバー/制約。

## Aha targets
- 新しい作業単位。
- 可逆なデフォルト。
- ゲーティングに使えるシグナル。

## Pitfalls & defusals
- Pitfall: 詩的解釈で終わる → Defusal: 具体レバーに翻訳できない案は捨てる。
- Pitfall: 小手先案になる → Defusal: 表層機能ではなく制約に対応付ける。

## Examples
### Engineering
Prompt: 「仕事をクリティカルパス外へ移す。」
Lever: 検証処理を非同期ワーカーへ移す。
Options: 非同期検証、段階適用、異常検知時のみ厳格化。

### Mixed domain
Prompt: 「努力ではなく調整を減らす。」
Lever: 会議を意思決定ログ中心の非同期へ置き換える。
Options: テンプレート、コメント期限、軽量ファシリテーション。
Signal: 意思決定サイクル時間。
Escape hatch: 明確性低下時は短い同期会議を再導入。
