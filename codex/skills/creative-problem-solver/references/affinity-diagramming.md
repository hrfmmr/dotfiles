# Technique: Affinity Diagramming (cluster → name themes)

## One-liner
発散後に、アイデアを似ているもの同士で黙ってまとめ、共通する意図で名前を付ける手法。散らかった案を、すぐ実行できる少数の方向に整理できる。

## Use when
- すでに20件以上の案があり、収束したい。
- 案の一覧がノイズっぽく、重複も多い。
- 実験すべき「テーマ」を見つけたい。

## Avoid when
- まだ案の幅が足りない（Brainstorming / Brainwriting を使う）。
- 大きな発想転換が必要（Synectics / Provocation を使う）。

## Inputs
- 生のアイデアの束（メモ、箇条書き）。

## Procedure (fast, 6–10 min)
1. アイデアを1行1案の粒度で書く。
2. 議論せず、似ている案を黙ってまとめる。
3. 各クラスタに「動詞+対象」で意図ラベルを付ける。
4. 各クラスタの代表案を1つ選び、実験案に磨く。

## Procedure (full, 15–25 min)
1. Atomicize
   - 複合的な案は、より小さなメモに分解する。
2. Silent clustering
   - クラスタが落ち着くまで、メモを移動して整える。
3. Naming
   - 「何か」ではなく「なぜ」で命名する（例: 「Slack」ではなく「引き継ぎを減らす」）。
4. Converge
   - クラスタを「シグナルの強さ + 可逆性」で評価する。
   - 3〜5クラスタを選び、各CPS tierに1実験ずつ対応させる。

## Prompt bank (copy/paste)
- 「このメモ群に共通する意図は何か？」
- 「このクラスタを1つの動詞で表すなら？」
- 「欠けているクラスタ（空白のテーマ）は何か？」

## Outputs (feed CPS portfolio)
- 3〜7個のテーマ。
- テーマごとに1つの具体化された候補。

## Aha targets
- 実装は違っても、意図は同じ案が多いと気づける。
- 抜けているテーマ（例: 観測性）が見える。

## Pitfalls & defusals
- Pitfall: クラスタ中に議論が始まる → Defusal: 命名が終わるまで沈黙を徹底する。
- Pitfall: 早すぎる命名 → Defusal: 先に分類、命名は後。

## Examples
### Engineering
入力: CI時間短縮の案40件。
クラスタ: 並列化、キャッシュ、対象範囲縮小、フレーク安定化、フィードバック改善。
各クラスタから1つ選んで実験化（シグナル: CI中央値、エスケープハッチ: キャッシュ不整合時は即ロールバック）。

### Mixed domain
入力: コミュニティ改善の案30件。
クラスタ: オンボーディング、儀式、モデレーション、承認、フィードバックループ。
各クラスタで1案を試す（シグナル: 継続率と参加の質、エスケープハッチ: 無理な儀式は中止）。
