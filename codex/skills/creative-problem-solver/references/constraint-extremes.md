# Technique: Constraint Extremes (0 / ∞ / absurd)

## One-liner
重要変数をゼロまたは無限に振って設計し、そこで見える不変原理を取り出して現実解へ戻す。局所最適から抜けるための方法。

## Use when
- 「現実的な制約」が曖昧、または思い込みの可能性がある。
- 「いつもこのやり方」で固定化している。
- 制約に根ざした発想の幅を作りたい。

## Avoid when
- 制約境界は十分わかっており、実行フェーズにいる。
- 問題が明確な矛盾（TRIZ）。

## Inputs
- 基本問題と、極端化する制約1〜2個。

## Procedure (fast, 5–10 min)
1. 変数を1つ選ぶ（時間、予算、リスク許容、人員、レイテンシ予算、ユーザー注意など）。
2. 2つの極端を回す。
   - Zero: 「これが0なら？」
   - Infinity: 「これが無限なら？」
3. 各極端で3案ずつ出す。
4. 不変条件を2〜3個抽出する。
5. 現実的なハイブリッド案を作り、実験に変換する。

## Procedure (full, 15–25 min)
1. 極端の設定を意図的に選ぶ。
   - 基本は 0 / ∞。
   - ほかに 1（単独担当）、10倍、1/10、可逆必須、監査必須も検討する。
2. 極端下で設計する。
   - 0側: 何を削る・簡素化する・自動化するか。
   - ∞側: 何を測る・専門化する・並列化するか。
3. 不変条件を抽出する。
   - 両極端で共通して必要なものは何か。
   - 片側だけで可能になるものは何か。
4. 補間する。
   - 不変条件を残しつつ、極端側のレバーを1つ借りた案を作る。
5. CPSポートフォリオに落とす。
   - 0側案はQuick Winになりやすく、∞側案はTransformative/Moonshotになりやすい。

## Prompt bank (copy/paste)
- 「予算0/時間0/人0でも、何は必ずやるか？」
- 「予算無限/時間無限なら、何を作るか？」
- 「両極端でも変わらないことは何か？」
- 「無限側案の最小スライスで試せるものは何か？」

## Outputs (feed CPS portfolio)
- 2つの極端設計。
- 不変原理のセット。
- 現実的ハイブリッド実験。

## Aha targets
- 本当の制約が金ではなく、調整コストや注意資源だと見える。
- 最適化すべき不変条件が明確になる。

## Pitfalls & defusals
- Pitfall: 空想で終わる → Defusal: 不変条件を抽出し、検証可能なハイブリッドへ戻す。
- Pitfall: 片方しか検討しない → Defusal: 0と∞の両方を必ず回す。
- Pitfall: 評価されないまま終わる → Defusal: シグナル + 可逆性で採点する。

## Examples
### Engineering
変数: ダウンタイム許容。
- ゼロダウンタイム: blue/green、canary、feature flag。
- 無限ダウンタイム: 最短移行 + 直接切替。
不変条件: ロールバック計画 + 観測性。
ハイブリッド: 1サービスで段階的canary。
シグナル: エラー率。
エスケープハッチ: ロールバックスイッチ。

### Mixed domain
変数: 使えるエネルギー。
- ゼロ: 手順削減、範囲縮小、通知自動化。
- 無限: 本格トレーニング、コーチ、計測。
不変条件: 強度より継続。
ハイブリッド: 小さな日次ルーチン + 週次レビュー。
シグナル: 継続率。
エスケープハッチ: ストレス上昇時は頻度を下げる。
